🔧 Reboot Credential Loss — Fix Guide

        

          

            
Step 1 — Disable Fast Startup

            
Control Panel > Power Options > Choose what the power buttons do > uncheck "Turn on fast startup". Then use Restart (not Shut Down) to test.

          

          

            
Step 2 — Re-add Credentials

            
Search "Credential Manager" in Start > Windows Credentials > Add a Windows credential. Enter your NAS address (\\NASname) and credentials. Check "Remember my credentials".

          

          

            
Step 3 — Check McAfee Firewall

            
McAfee can reset firewall rules for SMB (port 445) on reboot. Open McAfee > Firewall settings and verify File and Printer Sharing rules are set to Allow.

📘 How to get these values

          

            The TP-Link TL-SG2218 doesn't have a "username/password" for monitoring — it speaks
            SNMP. You need to enable SNMP on the switch and pick a community string.
          

          
    
            
  1. Find the switch IP: open a PowerShell window and run arp -a | findstr /i "dc-62-79" (the switch MAC starts with DC:62:79). Or check the Verizon router's Connected Devices page at 192.168.1.1.
    2. 
            
  2. Log into the switch web UI at http://<switch-ip> — factory default is admin / admin. If you forgot it, hold the reset pinhole 10 seconds to factory-reset.
    3. 
            
  3. Enable SNMP: navigate to SNMP in the sidebar (older firmware: System → SNMP → Global Config). Set SNMP to Enable and click Apply.
    4. 
            
  4. Add a community: go to SNMP → Community. Add or edit one with name public (or any alphanumeric string), Access read-only.
    5. 
            
  5. SAVE THE CONFIG: TP-Link gotcha — runtime changes vanish on reboot. Click Save Config (or System → System Tools → Config Backup → Save).
    6. 
            
  6. Come back here and fill in:
              
      
                
    • Switch IP Address: the IP from step 1 (or auto to auto-discover by MAC)
      • 
                
    • SNMP Community String: whatever name you set in step 4
      • 
              
    
            
    7. 
            
  7. Click Test to verify — success looks like "Connected! N/M ports up, X MACs learned."
    8. 
          

          
SNMP v2c with a community string is read-only and fine on a home LAN behind your edge router. Don't expose port 161 to the internet.
