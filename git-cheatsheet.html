⟳ Daily Workflow — 3 Commands to Save & Publish Changes
01
git add .
Stage all changed files — tells Git which changes to include in the next snapshot
→
02
git commit -m "message"
Save a snapshot locally with a short description of what changed
→
03
git push
Upload your commits to GitHub so they're backed up and visible online
Example commit messages: "Fix BIOS checker", "Add memory analysis tab", "Update README"
⚙ Setup — Run Once
git config --global user.name "Name"
Set your name for commits
git config --global user.email "email"
Set your email (use your GitHub email)
git init
Start tracking a folder with Git
git remote add origin URL
Connect local folder to a GitHub repo
git branch -M main
Rename branch to main
git push -u origin main
First push — sets up tracking
🔍 Check What's Going On
git status
See which files changed since last commit
git log --oneline
See commit history as a compact list
git log --oneline --graph
History with branch diagram
git diff
See exactly what lines changed (unstaged)
git diff --staged
See what's staged and ready to commit
git remote -v
Show connected GitHub URL
📦 Staging & Committing
git add .
Stage all changed files
git add filename.py
Stage one specific file
git restore --staged file
Unstage a file (undo git add)
git commit -m "message"
Save snapshot with a description
git commit --amend -m "new msg"
Fix the last commit message
git restore file
Discard changes to a file (undo edits)
☁ Push & Pull (GitHub Sync)
git push
Upload commits to GitHub
git pull
Download changes from GitHub
git pull --allow-unrelated-histories
Merge two separate histories (one-time fix)
git push -u origin main
Push and set tracking branch
git fetch
Download changes without merging yet
git clone URL
Download a repo from GitHub to your machine
↩ Undoing Things
git restore file
Discard unsaved changes to a file
git revert abc1234
Undo a commit safely (creates new commit)
git reset --soft HEAD~1
Undo last commit, keep changes staged
git reset --hard HEAD~1
⚠ Destructive — delete last commit + changes
Safest undo: always prefer git revert — it keeps history intact. Only use --hard reset if you're certain.

💡 Key Concepts
repository (repo) A project folder tracked by Git. Contains all your files + full history of every change.
commit A saved snapshot of your project at a point in time. Like a save game — you can always go back.
staging area A holding area for changes you're about to commit. git add puts files here.
branch A parallel version of your code. main is the primary branch. Create others to experiment safely.
remote The GitHub copy of your repo. "origin" is the standard name for it.
push / pull push = upload to GitHub. pull = download from GitHub. Always pull before pushing on shared repos.
.gitignore A file listing what Git should never track — passwords, cache files, logs, personal data.
HEAD A pointer to your current position in history. HEAD~1 means "one commit before current".
merge Combining two branches. git pull performs a merge automatically when remote has changes you don't.
Personal Access Token GitHub's substitute for a password when pushing from the command line. Generate at github.com/settings/tokens.
